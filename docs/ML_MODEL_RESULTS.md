## Deferral rationale

The GradientBoostingClassifier achieved F1=0.955 with 5-fold CV on 57 rows (~11 samples per fold). This is suspiciously high and likely reflects the clean decision boundaries in the synthetic seed data (42 of the 57 rows were generated by `seed_demo_data.py --seed-n 42`, which deterministically assigns DEAL to the first half and NO_DEAL to the second half). Logistic regression's ROC-AUC of 0.722 confirms real signal exists (price gaps predict outcomes), but the synthetic-only training set cannot justify shipping a production model.

**Rule applied** (from project's non-negotiable honesty constraint):
> "If data is still too thin for a meaningful CV, leave it deferred and say so explicitly rather than shipping an overfit model."

**The model artifact was generated during this run but then deleted.** It is not tracked in git and will not be deployed until:
1. Real LLM-negotiated sessions (data_source='real_llm_v6' or later) accumulate to at least 30 non-synthetic rows.
2. Retraining on mixed real + synthetic data shows the synthetic-to-real transfer gap is acceptably small.
3. A proper held-out test set (not just CV on training distribution) confirms the metrics.

## What this proves / does not prove

**Proves:** the feature engineering pipeline works end-to-end: `extract_features` runs, `StratifiedKFold` cross-validation completes, and the simpler logistic-regression baseline confirms that price gaps, convergence rates, and trust scores carry real predictive signal (ROC-AUC 0.722 > 0.5 baseline).

**Does not prove:** the model is ready for production. The 0.955 F1 on gradient boosting is inflated by synthetic data's clean labels. The model's generalization to real LLM-generated dialogues is unmeasured.
